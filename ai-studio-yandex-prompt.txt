5. ЯНДЕКС SDK И МОДЕРАЦИЯ (ОБЯЗАТЕЛЬНО)
═══════════════════════════════════════

Подключение:
- В index.html только: <script src="/sdk.js"></script>
- Без внешних CDN на Яндекс SDK.
- Порядок: init SDK → когда меню реально можно нажать «Играть» → LoadingAPI.ready().
- GameplayAPI.start/stop в правильных местах (старт раунда / пауза / меню / перед рекламой).

Реклама:
- Только через SDK.
- Любая кнопка, запускающая рекламу, ОБЯЗАНА явно предупреждать: текст «за видео» + заметная SVG-иконка телевизора/play.
- Во время interstitial/rewarded:
  gameState.isAdShowing = true
  полная блокировка кликов/тапов/геймплея
  звук выключен
- После закрытия рекламы: снять блок; звук вернуть только если !userMuted.

Звук:
- Ручной мьют хранить в gameState.userMuted (+ в сохранениях).
- visibilitychange / скрытие вкладки → всегда глушить.
- Возврат на вкладку / конец рекламы → звук только если пользователь сам не мьютил.

Сохранения:
- localStorage сразу после ключевых действий.
- Дублировать в облако через player.setData при тех же действиях (если player есть).
- Играть можно без авторизации; прогресс не теряется после F5.

Инапы (только если я сказал делать покупки):
- Только SDK payments + consumePurchase.
- Цена/валюта из каталога SDK.
- Список покупок в игре = покупки в консоли.

UI/платформы:
- Ориентацию фиксируем в концепте и везде одинаково.
- Desktop: игровое поле до краёв доступной области (не узкая колонка 480px).
- Без системного скролла, без contextmenu, touch-action: none.
- Без alert/confirm — только игровой UI.
- Игра выглядит законченной, не прототипом.
